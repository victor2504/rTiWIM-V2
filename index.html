RESUMO



O processo de levantamento de requisitos é uma das etapas fundamentais no ciclo de vida de um produto de software. No contexto de desenvolvimento de software, há cada vez mais a necessidade de ferramentas que apoiem os stakeholders no processo de elicitação e validação dos requisitos do sistema.

A participação do usuário final nesse processo, consolida a comunicação entre as partes interessadas para novas descobertas e problemáticas que o sistema possa apresentar. Dessa forma, a ferramenta denominada rTiWIM utilizará o modelo de comunicação TiWIM para elicitar e validar requisitos de software em sistemas web através da linguagem de interface.

A ferramenta irá dispor das seguintes funcionalidades: (i) cadastro de usuários, (ii) validação de páginas, (iii) cadastro de páginas para validação e (iv)validações realizadas por usuários finais. Além de um ambiente colaborativo para facilitar a comunicação entre os stakeholders .
